# QRコード作業管理アプリ（POC）要件定義

作成日: 2026-07-17

## 1. 概要

社員がスマートフォンでQRコードをスキャンし、製品の作業工程の進捗（開始・完了）をサーバPCへ記録するPOCアプリ。

## 2. QRコード体系

| 種別 | 数量 | 説明 |
|---|---|---|
| 社員番号QR | 100種類 | 社員1人につき1つ |
| 製品識別番号QR | 10種類 | 製品1つにつき1つ |
| 工程QR | 10製品 × 5種類 = 50種類 | 製品ごとに物理的に別のQRコード（コード内容に製品識別番号を含む） |

### 工程QRの内訳（1製品あたり5種類）

1. 作業工程1 開始
2. 作業工程1 完了
3. 作業工程2 開始
4. 作業工程2 完了
5. 全工程完了

### QRコード総数

100（社員） + 10（製品識別） + 50（工程） = **160種類**

テスト用には160種類全ては用意せず、動作確認に足る数種類のみ用意する。

## 3. 業務フロー・スキャン順序

1. 社員番号QR（ログイン相当、作業開始時に1回のみスキャン）
2. 製品識別番号QR
3. 作業工程1 開始QR
4. 作業工程1 完了QR
5. 作業工程2 開始QR
6. 作業工程2 完了QR
7. 全工程完了QR

- 手順②〜⑦は製品ごとに繰り返す（社員QRの再スキャンは不要）
- 上記順序通りでないスキャンはエラーとする（例：工程1完了を飛ばして工程2開始をスキャン等）

## 4. システム構成

- **Androidアプリ**: Webアプリ（PWA・ブラウザ）でカメラを使いQRコードをスキャン
- **通信**: 同一LAN内（自宅/社内Wi-Fi）でサーバPCへHTTP(S)送信
- **HTTPS対応**: AndroidブラウザのカメラAPI（getUserMedia）はHTTPS必須のため、mkcert等の自己署名証明書でサーバPCをローカルHTTPS化する
- **サーバPC**: Node.js（Express）+ SQLite
- **履歴確認**: ブラウザで一覧表示する簡易画面（社員・製品・工程ごとのスキャン履歴）
